# agile-refinement

Breaks large intakes or backlogs into executable stories with clear dependencies.

## When to use

- When a problem has been captured (intake) but still needs to be decomposed before execution
- When a backlog item is too large (L or XL) to execute directly
- When there is ambiguity about scope, dependencies, or order
- Before creating an epic with several stories
- After an intake that recommended refinement

## How to use

```
/agile-refinement
```

Example: `/agile-refinement auth-migration`

See [full documentation](../../docs/agile/agile-refinement.md) for end-to-end examples and tips.
