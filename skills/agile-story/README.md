# agile-story

Details a vertical delivery with acceptance criteria, tasks, and verification.

## When to use

- When the work has size M, involves several files, and needs richer acceptance and verification
- When a vertical delivery needs detailed acceptance criteria
- When a story from an epic needs to be detailed before execution
- When a bug fix has regression risk

## How to use

```
/agile-story
```

Example: `/agile-story user-login-flow`

See [full documentation](../../docs/agile/agile-story.md) for end-to-end examples and tips.
