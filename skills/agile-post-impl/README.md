# agile-post-impl

Generates post-implementation report with result, executed verifications, risks, and next steps.

## When to use

- When a delivery (plan, story, or epic) has been completed and needs to be formally closed
- When you need to compare what was planned vs what was delivered
- When you need to record verification results and remaining risks
- When closing a work cycle before retro

## How to use

```
/agile-post-impl
```

Example: `/agile-post-impl auth-refactor-plan`

See [full documentation](../../docs/agile/agile-post-impl.md) for end-to-end examples and tips.
