# agile-intake

Structures new and vague problems into clear intake documents.

## When to use

- When the problem is not yet mature enough for the backlog
- When someone brings an idea or need without defined scope
- When you need to decide what the next artifact in the flow should be
- When there is uncertainty about size, priority, or approach
- When it's the first contact with a new problem

## How to use

```
/agile-intake
```

Example: `/agile-intake component-tests`

See [full documentation](../../docs/agile/agile-intake.md) for end-to-end examples and tips.
