# agile-retro

Conducts retrospective with learnings and improvement actions.

## When to use

- When a cycle, sprint, or delivery has ended and the team needs to reflect on what worked and what needs to change
- When you need to separate facts from opinions and generate concrete actions
- When improvement actions need owners and deadlines
- After a sprint or delivery cycle closes

## How to use

```
/agile-retro
```

Example: `/agile-retro sprint-12`

See [full documentation](../../docs/agile/agile-retro.md) for end-to-end examples and tips.
