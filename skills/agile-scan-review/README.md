# agile-scan-review

Review changed code applying security, coherence, over-engineering, scope and quality checklists.

## When to use

- Before committing code
- When opening a PR
- When reviewing AI-generated code
- When you need a structured code review with quality gates

## How to use

```
/agile-scan-review
```

Example: `/agile-scan-review feature/auth`

See [full documentation](../../docs/agile/agile-scan-review.md) for end-to-end examples and tips.
