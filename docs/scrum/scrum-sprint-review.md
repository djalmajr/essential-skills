# Sprint Review

Prepara review e demo de entregas para stakeholders.

## Quando usar

- No fim de uma sprint
- Para apresentar resultado à equipe ou stakeholders
- Para validar que o produto está no caminho certo
- Coletar feedback para o próximo ciclo

## Quando NÃO usar

- Sprint ainda em andamento → aguarde
- Consolidado interno → `/status-report`
- Fechamento técnico de entrega → `/post-impl`

## Como usar

```
/sprint-review
```

Com contexto (sprint, período):

```
/sprint-review Sprint 23
/sprint-review Q2 2026 - payments
```

## O que o sprint review inclui

1. **Entregas consolidadas** — vs compromisso do planning
2. **Demo preparada** — por valor de negócio, não por tecnologia
3. **Itens não entregues** — com motivo e destino
4. **Mudanças de escopo** — registradas e explicadas
5. **Feedback dos stakeholders** — coletado para próximo ciclo

## Dica

Comece pelo **impacto**: "agora o usuário pode fazer X" é melhor que "implementamos a feature Y".

## Encadeamento

- Após sprint review → `/retro`
- Próxima sprint → `/sprint-planning`
